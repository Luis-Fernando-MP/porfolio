 Lenguaje de programación

<h2 className="notion-heading_2 notion-color-default">
 2. Patrones
</h2>


 3. Lexemas

  <ul className="notion-bulleted_list">

<li className="notion-bulleted_list_item notion-color-default">
 1&#95;variable → no es válido &#40;no empieza con letra o &#95;&#41;
</li>
  </ul>

 <b className="notion-text-bold">Análisis léxico:</b> 
</h3>
  
<p>
 Identifica y clasifica los elementos básicos del código fuente&#44; llamados <i className="notion-text-italic">lexemas</i>
&#44; y los convierte en unidades reconocibles llamadas <i className="notion-text-italic">tokens&#44; </i>
</p>
  <ul className="notion-bulleted_list">
<li className="notion-bulleted_list_item notion-color-default">
 Elimina espacios&#44; comentarios y otros elementos irrelevantes. Preparando la entrada para el análisis sintáctico.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 Además&#44; comprueba la correcta escritura de los símbolos&#44; palabras reservadas&#44; operadores&#44; etc. Y los agrupa según su tipo &#40;palabras reservadas&#44; identificadores&#44; literales&#44; etc.&#41;
</li>
  </ul>
<p>
 Un token léxico está constituido por una clave &#40;Nombre del token&#41; y un valor &#40;Valor del token&#41;. Algunos nombres comunes son:
</p>
  <ul className="notion-bulleted_list">
<li className="notion-bulleted_list_item notion-color-default">
 <b className="notion-text-bold">Identificadores</b>: nombres definidos por el programador.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 <b className="notion-text-bold">Palabras clave</b>: términos reservados por el lenguaje.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 <b className="notion-text-bold">Separadores</b>: signos de puntuación y delimitadores &#40;como: ; &#40;&#41; &#123;&#125;&#41;.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 <b className="notion-text-bold">Operadores</b>: símbolos que realizan operaciones &#40;&#43; -  / *&#41;.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 <b className="notion-text-bold">Literales</b>: valores constantes como números&#44; textos o booleanos.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 <b className="notion-text-bold">Comentarios</b>: líneas o bloques ignorados por el compilador; algunos se procesan como tokens&#44; otros se eliminan.
</li>
  </ul>
<h3 className="notion-heading_3 notion-color-default">
 Ejemplo:

 los tokens generados son:

<li className="notion-bulleted_list_item notion-color-default">

</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 Token 3: = Operador de asignación
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 Token 4: 5 Literal numérico

<li className="notion-bulleted_list_item notion-color-default">
 Token 5: &#43; Operador aritmético

<h3 className="notion-heading_3 notion-color-default">
 <b className="notion-text-bold">Análisis sintáctico: </b>
</h3>
  
<p>
 Se comprueba si lo obtenido de la fase anterior conforman estructuras sintácticamente correctas&#44; es decir si obedecen la gramática del lenguaje empleado. Por lo general&#44; las fases gramaticales del programa fuente se representan mediante un árbol de análisis sintáctico o un automata finito.

<h2 className="notion-heading_2 notion-color-default">
 <b className="notion-text-bold">🌲 Árboles de sintaxis abstracta:</b> 
</h2>
  
<p>
 Es una forma simplificada con base en un árbol que representa la estructura del código&#44; donde cada nodo muestra una construcción del programa sin incluir detalles como paréntesis o palabras clave.
</p>
  
<p>
 Ejemplo de un árbol de sintaxis abstracta&#44; usando como ejemplo en el Algoritmo de Euclides &#40;Es un método eficiente para calcular el máximo común divisor MCD de dos números enteros&#41;
</p>
  
<div className="notion-code">
 ```python
def MCD(a: int, b: int) -> int:
  """
  Calcula el Máximo Común Divisor usando el algoritmo de sustracción.
  """
  while b != 0:
 if a > b:
a = a - b
 else:
b = b - a
  return a
```

<h2 className="notion-heading_2 notion-color-default">
 Fase 2: Síntesis o Backend
</h2>
  
<p>
 Consiste en generar ya el<a href="https://es.wikipedia.org/wiki/C%C3%B3digo_objeto" className="notion-text-href"> </a>código objeto&#44; este paso solo se dará cuando el programa fuente está libre de errores de análisis.
</p>
  <ul className="notion-bulleted_list">
<li className="notion-bulleted_list_item notion-color-default">
 Las posiciones de memoria se seleccionan para cada una de las variables usadas por el programa. 
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 Cada una de las instrucciones intermedias se traduce a una secuencia de instrucciones de máquina que ejecuta la misma tarea.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
  Un aspecto decisivo es la asignación de variables a <a href="https://es.wikipedia.org/wiki/Registro_(hardware)" className="notion-text-href">registros</a>.
</li>
  </ul>
<h3 className="notion-heading_3 notion-color-default">
 Código intermedio
</h3>
  
<p>
 Se genera una representación intermedia explícita del programa fuente antes de generar el código objeto:
</p>
  <ul className="notion-bulleted_list">
<li className="notion-bulleted_list_item notion-color-default">
 debe ser fácil de producir y fácil de traducir al programa objeto
</li>
  </ul>
<p>
 La representación intermedia puede tener diversas formas. Existe una forma intermedia llamada código de tres direcciones&#44; similar al <a href="https://es.wikipedia.org/wiki/Lenguaje_ensamblador" className="notion-text-href">lenguaje ensamblador</a>
&#44; en la que cada instrucción lleva a cabo una sola operación. 
</p>
  
<h3 className="notion-heading_3 notion-color-default">
 Notación de tres direcciones &#40;T<b className="notion-text-bold">hree address code&#41;</b>
</h3>
  
<p>
 Es un lenguaje intermedio usado por compiladores optimizadores para ayudar en las transformaciones de mejora de código. 
</p>
  
<p>
 Ya que es usado como un lenguaje intermedio en los compiladores&#44; los operandos normalmente no contendrán direcciones de memoria o <a href="https://es.wikipedia.org/wiki/Registro_(hardware)" className="notion-text-href">registros</a> concretos&#44; sino que direcciones simbólicas que serán convertidas en direcciones reales durante la asignación de registros.
</p>
  
<p>
 Consiste en una secuencia de instrucciones&#44; sus propiedades son:
</p>
  <ul className="notion-bulleted_list">
<li className="notion-bulleted_list_item notion-color-default">
 <b className="notion-text-bold">Primera:</b> cada instrucción de tres direcciones tiene a lo sumo un operador&#44; además de la asignación.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 <b className="notion-text-bold">Segunda:</b> el traductor debe generar un nombre temporal para guardar los valores calculados por cada instrucción.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 <b className="notion-text-bold">Tercera: </b>algunas instrucciones «de tres direcciones» tienen menos de tres operandos.
</li>
  </ul>
<p>
 <b className="notion-text-bold">Ejemplo:</b>
</p>
  <ol className="notion-numbered_list">
<li className="notion-numbered_list_item notion-color-default">
 Ecuación cuadrática
</li>
  </ol>
<div className="notion-code">
 ```plain
x = (-b + sqrt(b^2 - 4*a*c)) / (2*a)

#Salida:
t1 := b * b
t2 := 4 * a
t3 := t2 * c
t4 := t1 - t3
t5 := sqrt(t4)
t6 := 0 - b
t7 := t5 + t6
t8 := 2 * a
t9 := t7 / t8
x := t9
```
 
</div>
  <ol className="notion-numbered_list">
<li className="notion-numbered_list_item notion-color-default">
 Bucle
</li>
  </ol>
<div className="notion-code">
 ```plain
for (i = 0; i < 10; ++i) {
 b[i] = i*i; 
}
#Salida:
  t1 := 0 ; inicializa i
L1:  if t1 >= 10 goto L2 ; salto condicional
  t2 := t1 * t1 ; cuadrado de i
  t3 := t1 * 4  ; reserva de cuatro bytes para el almacenamiento
  t4 := b + t3  ; dirección para almacenar i*i
  *t4 := t2  ; almacenar mediante puntero
  t1 := t1 + 1  ; incrementar i
  goto L1 ; repetir bucle
L2:
```
 
</div>

 Optimización de código
</h3>
  
<p>
 Consiste en mejorar el código intermedio&#44; de modo que resulte un código máquina más rápido de ejecutar:
</p>
  <ol className="notion-numbered_list">
<li className="notion-numbered_list_item notion-color-default">
 Eliminacion de codigo muerto
</li>
  
<li className="notion-numbered_list_item notion-color-default">
 Propagacion de constantes

<li className="notion-numbered_list_item notion-color-default">
 Eliminacion de subexpresiones comunes
</li>
  </ol>
<h2 className="notion-heading_2 notion-color-default">
 Fase 2: Síntesis o Backend
</h2>
  
<p>
 Esta fase toma el código intermedio y lo convierte en <b className="notion-text-bold">código objeto</b> &#40;también llamado código de máquina o ensamblador&#41;. Pero <b className="notion-text-bold">solo se ejecuta si el programa ya pasó todas las fases de análisis sin errores</b>.
</p>
  
<h3 className="notion-heading_3 notion-color-default">
 Tareas que realiza :
</h3>
  
<table className="notion-table">
 
<tr className="notion-table_row">
 <td className="notion-table_cell"><b className="notion-text-bold">Asignación de memoria</b></td><td className="notion-table_cell">Se reserva espacio en memoria para todas las variables del programa.</td>
</tr>
  
<tr className="notion-table_row">
 <td className="notion-table_cell"><b className="notion-text-bold">Traducción a instrucciones</b></td><td className="notion-table_cell">Cada instrucción del código intermedio se transforma en una o varias instrucciones de máquina reales.</td>
</tr>
  
<tr className="notion-table_row">
 <td className="notion-table_cell"><b className="notion-text-bold">Asignación de registros</b></td><td className="notion-table_cell">Se intenta usar registros del procesador &#40;más rápidos&#41; para almacenar variables temporales.</td>
</tr>
  
</table>
  
<h3 className="notion-heading_3 notion-color-default">
 Representación Intermedia
</h3>
  
<p>
 Antes de generar el código máquina&#44; se genera una <b className="notion-text-bold">versión intermedia del programa</b>
&#44; que:
</p>
  <ul className="notion-bulleted_list">
<li className="notion-bulleted_list_item notion-color-default">
 Sea <b className="notion-text-bold">fácil de analizar y optimizar</b>.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 No esté atada a una máquina específica.
</li>
  </ul>
<p>
 Para ello emplea diferentes formas&#44; la mas popular es el <b className="notion-text-bold">código de tres direcciones</b>.
</p>
  
<h3 className="notion-heading_3 notion-color-default">
 Notación de Tres Direcciones &#40;Three Address Code - TAC&#41;
</h3>
  
<p>
 Es un lenguaje intermedio donde: &#44; y guarda el resultado en una variable temporal:
</p>
  <ul className="notion-bulleted_list">
<li className="notion-bulleted_list_item notion-color-default">
 Cada instrucción hace una sola operación entre como máximo dos operandos.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 Cada resultado intermedio se guarda en una variable temporal para poder usarse después.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 Puede tener menos de tres operandos: Por ejemplo t1 := -b
</li>
  </ul><ol className="notion-numbered_list">
<li className="notion-numbered_list_item notion-color-default">
 <b className="notion-text-bold">Fórmula cuadrática</b>
</li>
  </ol>
<div className="notion-code">
 ```c
x = (-b + sqrt(b^2 - 4*a*c)) / (2*a)
```
 
</div>
  
<p>
 <b className="notion-text-bold">Código en tres direcciones</b>
</p>
  
<div className="notion-code">
 ```plain
t1 := b * b
t2 := 4 * a
t3 := t2 * c
t4 := t1 - t3
t5 := sqrt(t4)
t6 := 0 - b
t7 := t5 + t6
t8 := 2 * a
t9 := t7 / t8
```
 
</div>
  <ol className="notion-numbered_list">
<li className="notion-numbered_list_item notion-color-default">
 <b className="notion-text-bold">Bucle</b>
</li>
  </ol>
<div className="notion-code">
 ```c
for (i = 0; i < 10; i++) {
 b[i] = i * i;
}
```
 
</div>
  
<p>
 <b className="notion-text-bold">Código en tres direcciones:</b>
</p>
  
<div className="notion-code">
 ```plain
t1 := 0 ; i = 0
L1: if t1 >= 10 goto L2
t2 := t1 * t1
t3 := t1 * 4  ; para acceder al arreglo (4 bytes por int)
t4 := b + t3  ; puntero a b[i]
*t4 := t2  ; asigna valor a b[i]
t1 := t1 + 1
goto L1
L2:
```

<h3 className="notion-heading_3 notion-color-default">
 Optimización de Código
</h3>
  
<p>
 Antes de generar el código final&#44; el compilador mejora el intermedio con técnicas de <b className="notion-text-bold">optimización</b>:
</p>
  <ol className="notion-numbered_list">
<li className="notion-numbered_list_item notion-color-default">
 <b className="notion-text-bold">Eliminación de código muerto</b>
</li>
  </ol>
<div className="notion-code">
 ```c
x = 10;
y = 20;

// Salida
y = 20;
x = 15;
print(x);
```
 
</div>
  <ol className="notion-numbered_list">
<li className="notion-numbered_list_item notion-color-default">
 <b className="notion-text-bold">Propagación de constantes</b>
</li>
  </ol>
<p>
 Sustituye variables que contienen valores constantes directamente por ese valor en todo el código
</p>
  
<div className="notion-code">
 ```c
const int a = 3;
int b = a + 2;
int c = b * a;

//Salida
int b = 3 + 2;// se reemplaza 'a'

```
 
</div>
  <ol className="notion-numbered_list">
<li className="notion-numbered_list_item notion-color-default">
 <b className="notion-text-bold">Eliminación de subexpresiones comunes</b>
</li>
  </ol>
<p>
 Detecta y reutiliza cálculos repetidos para evitar hacerlos de nuevo.
</p>
  
<div className="notion-code">
 ```c
x = a * b + c;
y = a * b + d;

// Salida
t1 = a * b;
x = t1 + c;
y = t1 + d;
```
 
</div>
  
<p>

&lt;h1 class=&quot;notion-heading&#95;1 notion-color-default&quot;&gt;
 Lenguaje de programación
&lt;/h1&gt;
  
&lt;p&gt;
 Es un &lt;b class=&quot;notion-text-bold&quot;&gt;sistema formal&lt;/b&gt; con el que le damos instrucciones a una computadora para que realice tareas. Tiene:
&lt;/p&gt;
  &lt;ul class=&quot;notion-bulleted&#95;list&quot;&gt;
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Sintaxis&lt;/b&gt;: Reglas de cómo se escriben las instrucciones &#40;como la gramática en español&#41;.
&lt;/li&gt;
  
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Semántica&lt;/b&gt;: El significado de las instrucciones.
&lt;/li&gt;
  &lt;/ul&gt;

&lt;h2 class=&quot;notion-heading&#95;2 notion-color-default&quot;&gt;
 Fase 1: Análisis o Frontend
&lt;/h2&gt;
  
&lt;p&gt;
 Busca la comprobación y corrección código fuente&#44; descomponiéndolo en estructuras comprensibles para el compilador. Según la definición del “lenguaje en términos de teoría de lenguajes formales”&#44; se divide en: 
&lt;/p&gt;
  
&lt;h3 class=&quot;notion-heading&#95;3 notion-color-default&quot;&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Análisis léxico:&lt;/b&gt; 
&lt;/h3&gt;
  
&lt;p&gt;
 Identifica y clasifica los elementos básicos del código fuente&#44; llamados &lt;i class=&quot;notion-text-italic&quot;&gt;lexemas&lt;/i&gt;&#44; y los convierte en unidades reconocibles llamadas &lt;i class=&quot;notion-text-italic&quot;&gt;tokens&#44; &lt;/i&gt;
&lt;/p&gt;
  &lt;ul class=&quot;notion-bulleted&#95;list&quot;&gt;
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 Elimina espacios&#44; comentarios y otros elementos irrelevantes. Preparando la entrada para el análisis sintáctico.
&lt;/li&gt;
  
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 Además&#44; comprueba la correcta escritura de los símbolos&#44; palabras reservadas&#44; operadores&#44; etc. Y los agrupa según su tipo &#40;palabras reservadas&#44; identificadores&#44; literales&#44; etc.&#41;
&lt;/li&gt;
  &lt;/ul&gt;
&lt;p&gt;
 Un token léxico está constituido por una clave &#40;Nombre del token&#41; y un valor &#40;Valor del token&#41;. Algunos nombres comunes son:
&lt;/p&gt;
  &lt;ul class=&quot;notion-bulleted&#95;list&quot;&gt;
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Identificadores&lt;/b&gt;: nombres definidos por el programador.
&lt;/li&gt;
  
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Palabras clave&lt;/b&gt;: términos reservados por el lenguaje.
&lt;/li&gt;
  
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Separadores&lt;/b&gt;: signos de puntuación y delimitadores &#40;como: ; &#40;&#41; &#123;&#125;&#41;.
&lt;/li&gt;
  
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Operadores&lt;/b&gt;: símbolos que realizan operaciones &#40;&#43; -  / *&#41;.
&lt;/li&gt;
  
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Literales&lt;/b&gt;: valores constantes como números&#44; textos o booleanos.
&lt;/li&gt;
  
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Comentarios&lt;/b&gt;: líneas o bloques ignorados por el compilador; algunos se procesan como tokens&#44; otros se eliminan.
&lt;/li&gt;
  &lt;/ul&gt;
&lt;h3 class=&quot;notion-heading&#95;3 notion-color-default&quot;&gt;
 Ejemplo:
&lt;/h3&gt;
  
&lt;div class=&quot;notion-code&quot;&gt;
 &lt;pre&gt;&lt;code class=&quot;language-python&quot;&gt;int suma = 5 &#43; 3;&lt;/code&gt;&lt;/pre&gt;

&lt;h3 class=&quot;notion-heading&#95;3 notion-color-default&quot;&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Análisis sintáctico: &lt;/b&gt;
&lt;/h3&gt;
  
&lt;p&gt;
 Se comprueba si lo obtenido de la fase anterior conforman estructuras sintácticamente correctas&#44; es decir si obedecen la gramática del lenguaje empleado. Por lo general&#44; las fases gramaticales del programa fuente se representan mediante un árbol de análisis sintáctico o un automata finito. 
&lt;/p&gt;
  
&lt;h2 class=&quot;notion-heading&#95;2 notion-color-default&quot;&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;🌲 Árboles de sintaxis abstracta:&lt;/b&gt; 
&lt;/h2&gt;
  
&lt;p&gt;
 Es una forma simplificada con base en un árbol que representa la estructura del código&#44; donde cada nodo muestra una construcción del programa sin incluir detalles como paréntesis o palabras clave.
&lt;/p&gt;
  
&lt;p&gt;
 Ejemplo de un árbol de sintaxis abstracta&#44; usando como ejemplo en el Algoritmo de Euclides &#40;Es un método eficiente para calcular el máximo común divisor MCD de dos números enteros&#41;
&lt;/p&gt;

  &quot;&quot;&quot;
  Calcula el M&#225;ximo Com&#250;n Divisor usando el algoritmo de sustracci&#243;n.
  &quot;&quot;&quot;
  while b != 0:
 if a &#62; b:
a = a - b
 else:
b = b - a
  return a&lt;/code&gt;&lt;/pre&gt;

&lt;/h3&gt;

&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 Revisa que las variables estén declaradas y usadas correctamente.
&lt;/li&gt;
  
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 Asegura que las funciones&#44; arreglos&#44; condicionales&#44; etc.&#44; tengan sentido según el contexto.
&lt;/li&gt;

&lt;h2 class=&quot;notion-heading&#95;2 notion-color-default&quot;&gt;
 Fase 2: Síntesis o Backend
&lt;/h2&gt;
  
&lt;p&gt;
 Consiste en generar ya el&lt;a href=&quot;https://es.wikipedia.org/wiki/C%C3%B3digo&#95;objeto&quot; class=&quot;notion-text-href&quot;&gt; &lt;/a&gt;código objeto&#44; este paso solo se dará cuando el programa fuente está libre de errores de análisis.
&lt;/p&gt;
  &lt;ul class=&quot;notion-bulleted&#95;list&quot;&gt;
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 Las posiciones de memoria se seleccionan para cada una de las variables usadas por el programa. 
&lt;/li&gt;
  
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 Cada una de las instrucciones intermedias se traduce a una secuencia de instrucciones de máquina que ejecuta la misma tarea.
&lt;/li&gt;
  
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
  Un aspecto decisivo es la asignación de variables a &lt;a href=&quot;https://es.wikipedia.org/wiki/Registro&#95;&#40;hardware&#41;&quot; class=&quot;notion-text-href&quot;&gt;registros&lt;/a&gt;.
&lt;/li&gt;
  &lt;/ul&gt;
&lt;h3 class=&quot;notion-heading&#95;3 notion-color-default&quot;&gt;
 Código intermedio
&lt;/h3&gt;
  
&lt;p&gt;
 Se genera una representación intermedia explícita del programa fuente antes de generar el código objeto:
&lt;/p&gt;
  &lt;ul class=&quot;notion-bulleted&#95;list&quot;&gt;
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 debe ser fácil de producir y fácil de traducir al programa objeto
&lt;/li&gt;
  &lt;/ul&gt;
&lt;p&gt;
 La representación intermedia puede tener diversas formas. Existe una forma intermedia llamada código de tres direcciones&#44; similar al &lt;a href=&quot;https://es.wikipedia.org/wiki/Lenguaje&#95;ensamblador&quot; class=&quot;notion-text-href&quot;&gt;lenguaje ensamblador&lt;/a&gt;&#44; en la que cada instrucción lleva a cabo una sola operación. 
&lt;/p&gt;
  
&lt;h3 class=&quot;notion-heading&#95;3 notion-color-default&quot;&gt;
 Notación de tres direcciones &#40;T&lt;b class=&quot;notion-text-bold&quot;&gt;hree address code&#41;&lt;/b&gt;
&lt;/h3&gt;
  
&lt;p&gt;
 Es un lenguaje intermedio usado por compiladores optimizadores para ayudar en las transformaciones de mejora de código. 
&lt;/p&gt;
  
&lt;p&gt;
 Ya que es usado como un lenguaje intermedio en los compiladores&#44; los operandos normalmente no contendrán direcciones de memoria o &lt;a href=&quot;https://es.wikipedia.org/wiki/Registro&#95;&#40;hardware&#41;&quot; class=&quot;notion-text-href&quot;&gt;registros&lt;/a&gt; concretos&#44; sino que direcciones simbólicas que serán convertidas en direcciones reales durante la asignación de registros.
&lt;/p&gt;
  
&lt;p&gt;
 Consiste en una secuencia de instrucciones&#44; sus propiedades son:
&lt;/p&gt;
  &lt;ul class=&quot;notion-bulleted&#95;list&quot;&gt;
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Primera:&lt;/b&gt; cada instrucción de tres direcciones tiene a lo sumo un operador&#44; además de la asignación.
&lt;/li&gt;
  
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Segunda:&lt;/b&gt; el traductor debe generar un nombre temporal para guardar los valores calculados por cada instrucción.
&lt;/li&gt;
  
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Tercera: &lt;/b&gt;algunas instrucciones «de tres direcciones» tienen menos de tres operandos.
&lt;/li&gt;
  &lt;/ul&gt;
&lt;p&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Ejemplo:&lt;/b&gt;
&lt;/p&gt;
  &lt;ol class=&quot;notion-numbered&#95;list&quot;&gt;
&lt;li class=&quot;notion-numbered&#95;list&#95;item notion-color-default&quot;&gt;
 Ecuación cuadrática
&lt;/li&gt;
  &lt;/ol&gt;
&lt;div class=&quot;notion-code&quot;&gt;
 &lt;pre&gt;&lt;code class=&quot;language-plain text&quot;&gt;x = &#40;-b &#43; sqrt&#40;b&#94;2 - 4*a*c&#41;&#41; / &#40;2*a&#41;

&#35;Salida:
t1 := b * b
t2 := 4 * a
t3 := t2 * c
t4 := t1 - t3
t5 := sqrt&#40;t4&#41;
t6 := 0 - b
t7 := t5 &#43; t6
t8 := 2 * a
t9 := t7 / t8
x := t9&lt;/code&gt;&lt;/pre&gt;
 
&lt;/div&gt;
  &lt;ol class=&quot;notion-numbered&#95;list&quot;&gt;
&lt;li class=&quot;notion-numbered&#95;list&#95;item notion-color-default&quot;&gt;
 Bucle
&lt;/li&gt;
  &lt;/ol&gt;
&lt;div class=&quot;notion-code&quot;&gt;
 &lt;pre&gt;&lt;code class=&quot;language-plain text&quot;&gt;for &#40;i = 0; i &#60; 10; &#43;&#43;i&#41; &#123;
 b[i] = i*i; 
&#125;
&#35;Salida:
  t1 := 0 ; inicializa i
L1:  if t1 &#62;= 10 goto L2 ; salto condicional
  t2 := t1 * t1 ; cuadrado de i
  t3 := t1 * 4  ; reserva de cuatro bytes para el almacenamiento
  t4 := b &#43; t3  ; direcci&#243;n para almacenar i*i
  *t4 := t2  ; almacenar mediante puntero
  t1 := t1 &#43; 1  ; incrementar i
  goto L1 ; repetir bucle
L2:&lt;/code&gt;&lt;/pre&gt;
 
&lt;/div&gt;
  
&lt;h3 class=&quot;notion-heading&#95;3 notion-color-default&quot;&gt;
 Optimización de código
&lt;/h3&gt;
  
&lt;p&gt;
 Consiste en mejorar el código intermedio&#44; de modo que resulte un código máquina más rápido de ejecutar:
&lt;/p&gt;
  &lt;ol class=&quot;notion-numbered&#95;list&quot;&gt;
&lt;li class=&quot;notion-numbered&#95;list&#95;item notion-color-default&quot;&gt;
 Eliminacion de codigo muerto
&lt;/li&gt;
  
&lt;li class=&quot;notion-numbered&#95;list&#95;item notion-color-default&quot;&gt;
 Propagacion de constantes
&lt;/li&gt;
  
&lt;li class=&quot;notion-numbered&#95;list&#95;item notion-color-default&quot;&gt;
 Eliminacion de subexpresiones comunes
&lt;/li&gt;
  &lt;/ol&gt;
&lt;h2 class=&quot;notion-heading&#95;2 notion-color-default&quot;&gt;
 Fase 2: Síntesis o Backend
&lt;/h2&gt;
  
&lt;p&gt;
 Esta fase toma el código intermedio y lo convierte en &lt;b class=&quot;notion-text-bold&quot;&gt;código objeto&lt;/b&gt; &#40;también llamado código de máquina o ensamblador&#41;. Pero &lt;b class=&quot;notion-text-bold&quot;&gt;solo se ejecuta si el programa ya pasó todas las fases de análisis sin errores&lt;/b&gt;.
&lt;/p&gt;
  
&lt;h3 class=&quot;notion-heading&#95;3 notion-color-default&quot;&gt;
 Tareas que realiza :
&lt;/h3&gt;
  
&lt;table class=&quot;notion-table&quot;&gt;
 
&lt;tr class=&quot;notion-table&#95;row&quot;&gt;
 &lt;td class=&quot;notion-table&#95;cell&quot;&gt;&lt;b class=&quot;notion-text-bold&quot;&gt;Asignación de memoria&lt;/b&gt;&lt;/td&gt;&lt;td class=&quot;notion-table&#95;cell&quot;&gt;Se reserva espacio en memoria para todas las variables del programa.&lt;/td&gt;
&lt;/tr&gt;
  
&lt;tr class=&quot;notion-table&#95;row&quot;&gt;
 &lt;td class=&quot;notion-table&#95;cell&quot;&gt;&lt;b class=&quot;notion-text-bold&quot;&gt;Traducción a instrucciones&lt;/b&gt;&lt;/td&gt;&lt;td class=&quot;notion-table&#95;cell&quot;&gt;Cada instrucción del código intermedio se transforma en una o varias instrucciones de máquina reales.&lt;/td&gt;
&lt;/tr&gt;
  
&lt;tr class=&quot;notion-table&#95;row&quot;&gt;
 &lt;td class=&quot;notion-table&#95;cell&quot;&gt;&lt;b class=&quot;notion-text-bold&quot;&gt;Asignación de registros&lt;/b&gt;&lt;/td&gt;&lt;td class=&quot;notion-table&#95;cell&quot;&gt;Se intenta usar registros del procesador &#40;más rápidos&#41; para almacenar variables temporales.&lt;/td&gt;

&lt;h3 class=&quot;notion-heading&#95;3 notion-color-default&quot;&gt;
 Representación Intermedia
&lt;/h3&gt;
  
&lt;p&gt;
 Antes de generar el código máquina&#44; se genera una &lt;b class=&quot;notion-text-bold&quot;&gt;versión intermedia del programa&lt;/b&gt;&#44; que:
&lt;/p&gt;
  &lt;ul class=&quot;notion-bulleted&#95;list&quot;&gt;
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 Sea &lt;b class=&quot;notion-text-bold&quot;&gt;fácil de analizar y optimizar&lt;/b&gt;.
&lt;/li&gt;
  
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 No esté atada a una máquina específica.
&lt;/li&gt;
  &lt;/ul&gt;
&lt;p&gt;
 Para ello emplea diferentes formas&#44; la mas popular es el &lt;b class=&quot;notion-text-bold&quot;&gt;código de tres direcciones&lt;/b&gt;.
&lt;/p&gt;
  
&lt;h3 class=&quot;notion-heading&#95;3 notion-color-default&quot;&gt;
 Notación de Tres Direcciones &#40;Three Address Code - TAC&#41;
&lt;/h3&gt;
  
&lt;p&gt;
 Es un lenguaje intermedio donde: &#44; y guarda el resultado en una variable temporal:
&lt;/p&gt;
  &lt;ul class=&quot;notion-bulleted&#95;list&quot;&gt;
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 Cada instrucción hace una sola operación entre como máximo dos operandos.
&lt;/li&gt;
  
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 Cada resultado intermedio se guarda en una variable temporal para poder usarse después.
&lt;/li&gt;
  
&lt;li class=&quot;notion-bulleted&#95;list&#95;item notion-color-default&quot;&gt;
 Puede tener menos de tres operandos: Por ejemplo t1 := -b
&lt;/li&gt;
  &lt;/ul&gt;&lt;ol class=&quot;notion-numbered&#95;list&quot;&gt;
&lt;li class=&quot;notion-numbered&#95;list&#95;item notion-color-default&quot;&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Fórmula cuadrática&lt;/b&gt;
&lt;/li&gt;
  &lt;/ol&gt;
&lt;div class=&quot;notion-code&quot;&gt;
 &lt;pre&gt;&lt;code class=&quot;language-c&quot;&gt;x = &#40;-b &#43; sqrt&#40;b&#94;2 - 4*a*c&#41;&#41; / &#40;2*a&#41;&lt;/code&gt;&lt;/pre&gt;
 
&lt;/div&gt;
  
&lt;p&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Código en tres direcciones&lt;/b&gt;
&lt;/p&gt;
  
&lt;div class=&quot;notion-code&quot;&gt;
 &lt;pre&gt;&lt;code class=&quot;language-plain text&quot;&gt;t1 := b * b
t2 := 4 * a
t3 := t2 * c
t4 := t1 - t3
t5 := sqrt&#40;t4&#41;
t6 := 0 - b
t7 := t5 &#43; t6
t8 := 2 * a
t9 := t7 / t8&lt;/code&gt;&lt;/pre&gt;
 
&lt;/div&gt;
  &lt;ol class=&quot;notion-numbered&#95;list&quot;&gt;
&lt;li class=&quot;notion-numbered&#95;list&#95;item notion-color-default&quot;&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Bucle&lt;/b&gt;
&lt;/li&gt;
  &lt;/ol&gt;
&lt;div class=&quot;notion-code&quot;&gt;
 &lt;pre&gt;&lt;code class=&quot;language-c&quot;&gt;for &#40;i = 0; i &#60; 10; i&#43;&#43;&#41; &#123;
 b[i] = i * i;
&#125;&lt;/code&gt;&lt;/pre&gt;
 
&lt;/div&gt;
  
&lt;p&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Código en tres direcciones:&lt;/b&gt;
&lt;/p&gt;
  
&lt;div class=&quot;notion-code&quot;&gt;
 &lt;pre&gt;&lt;code class=&quot;language-plain text&quot;&gt;t1 := 0 ; i = 0
L1: if t1 &#62;= 10 goto L2
t2 := t1 * t1
t3 := t1 * 4  ; para acceder al arreglo &#40;4 bytes por int&#41;
t4 := b &#43; t3  ; puntero a b[i]
*t4 := t2  ; asigna valor a b[i]
t1 := t1 &#43; 1
goto L1
L2:&lt;/code&gt;&lt;/pre&gt;
 
&lt;/div&gt;
  
&lt;h3 class=&quot;notion-heading&#95;3 notion-color-default&quot;&gt;
 Optimización de Código
&lt;/h3&gt;
  
&lt;p&gt;
 Antes de generar el código final&#44; el compilador mejora el intermedio con técnicas de &lt;b class=&quot;notion-text-bold&quot;&gt;optimización&lt;/b&gt;:
&lt;/p&gt;
  &lt;ol class=&quot;notion-numbered&#95;list&quot;&gt;
&lt;li class=&quot;notion-numbered&#95;list&#95;item notion-color-default&quot;&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Eliminación de código muerto&lt;/b&gt;
&lt;/li&gt;
  &lt;/ol&gt;
&lt;div class=&quot;notion-code&quot;&gt;
 &lt;pre&gt;&lt;code class=&quot;language-c&quot;&gt;x = 10;
y = 20;
x = 15;  &#47;&#47;la asignaci&#243;n anterior a x = 10 es in&#250;til
print&#40;x&#41;;

&#47;&#47;Salida
y = 20;
x = 15;
print&#40;x&#41;;&lt;/code&gt;&lt;/pre&gt;
 
&lt;/div&gt;
  &lt;ol class=&quot;notion-numbered&#95;list&quot;&gt;
&lt;li class=&quot;notion-numbered&#95;list&#95;item notion-color-default&quot;&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Propagación de constantes&lt;/b&gt;
&lt;/li&gt;
  &lt;/ol&gt;
&lt;p&gt;
 Sustituye variables que contienen valores constantes directamente por ese valor en todo el código
&lt;/p&gt;
  
&lt;div class=&quot;notion-code&quot;&gt;
 &lt;pre&gt;&lt;code class=&quot;language-c&quot;&gt;const int a = 3;
int b = a &#43; 2;
int c = b * a;

//Salida
int b = 3 &#43; 2;&#47;&#47;se reemplaza &#39;a&#39;
int c = b * 3;&#47;&#47;tambi&#233;n aqu&#237;&lt;/code&gt;&lt;/pre&gt;
 
&lt;/div&gt;
  &lt;ol class=&quot;notion-numbered&#95;list&quot;&gt;
&lt;li class=&quot;notion-numbered&#95;list&#95;item notion-color-default&quot;&gt;
 &lt;b class=&quot;notion-text-bold&quot;&gt;Eliminación de subexpresiones comunes&lt;/b&gt;
&lt;/li&gt;
  &lt;/ol&gt;
&lt;p&gt;
 Detecta y reutiliza cálculos repetidos para evitar hacerlos de nuevo.
&lt;/p&gt;
  
&lt;div class=&quot;notion-code&quot;&gt;
 &lt;pre&gt;&lt;code class=&quot;language-c&quot;&gt;x = a * b &#43; c;
y = a * b &#43; d;

&#47;&#47;Salida
t1 = a * b;
x = t1 &#43; c;
y = t1 &#43; d;&lt;/code&gt;&lt;/pre&gt;
 
&lt;/div&gt;
  
&lt;p&gt;
 
&lt;/p&gt;

<p>
 Este tipo de framework nos permiten construir diseños directamente en el markup &#40;HTML&#41;&#44; utilizando clases atómicas que representan propiedades CSS individuales. A diferencia de los frameworks tradicionales que nos ofrecen componentes predefinidos&#44; los frameworks utility-first nos dan las herramientas básicas para construir nuestros propios componentes desde cero.
</p>
  
<h3 className="notion-heading_3 notion-color-default">
 Ventajas:
</h3>
  <ul className="notion-bulleted_list">
<li className="notion-bulleted_list_item notion-color-default">
 Tiene una flexibilidad absoluta&#44; ya que nosotros podemos crear cualquier diseño sin salir del HTML&#44; lo que acelera significativamente el proceso de desarrollo. 
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 Evitamos el problema de los nombres de clases y la especificidad en CSS. 
</li>
  </ul><ul className="notion-bulleted_list">
<li className="notion-bulleted_list_item notion-color-default">
 Hay más verbosidad en el HTML lo que puede ser abrumadora al principio
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 La curva de aprendizaje puede ser más pronunciada&#44; aunque al inicio puede ser abrumador&#44; se vuelve más natural para quienes dominan la lógica de CSS nativo
</li>
  </ul>
<p>
 Es la encarnación más popular del enfoque utility-first. Se define como un framework altamente personalizable que nos proporciona clases de bajo nivel para construir diseños complejos. Tailwind no impone estilos predefinidos&#44; sino que ofrece los bloques o clases atómicas de construcción fundamentales para crear casi cualquier diseño.
</p>
  
<h3 className="notion-heading_3 notion-color-default">
 Características:
</h3>
  <ul className="notion-bulleted_list">
<li className="notion-bulleted_list_item notion-color-default">
 Su sistema de diseño nos ayuda a trabajar con una paleta de colores consistente&#44; una escala de espaciado predefinida&#44; un sistema de tipografía coherente y demás. Estas características nos permiten mantener la consistencia en todo nuestro proyecto sin ningún esfuerzo.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 Se puede configurar fácilmente cada aspecto del framework para adaptarlo a nuestras necesidades específicas&#44; desde la paleta de colores hasta los breakpoints del responsive design.
</li>
  </ul>

<h1 className="notion-heading_1 notion-color-default">
 <b className="notion-text-bold">¿Qué son los </b><b className="notion-text-bold">&lt;span className=&quot;notion-color-brown&quot;&gt;Framework Intermedios&lt;/span&gt;</b><b className="notion-text-bold">
&#63;</b>
</h1>
  
<p>
 Estos frameworks nos ofrecen un equilibrio entre control de estilo y componentes útiles. Incluyen grillas&#44; botones&#44; tarjetas y formularios listos para usar&#44; sin imponer un diseño rígido. Son flexibles y menos verbosos que los Frameworks completos&#44; y no nos obligan a usar JavaScript.
</p>
  
<h3 className="notion-heading_3 notion-color-default">
 Ventajas:
</h3>
  <ul className="notion-bulleted_list">
<li className="notion-bulleted_list_item notion-color-default">
 Diseño limpio y minimalista.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 Flexibilidad visual &#40;con opciones CSS o Sass&#41;.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 Estructura modular: incluimos solo lo que necesitamos.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 Sintaxis semántica y fácil de aprender &#40;a diferencia de Tailwind CSS&#41;
</li>
  </ul><ul className="notion-bulleted_list">
<li className="notion-bulleted_list_item notion-color-default">
 No incluyen interactividad por defecto; si la necesitamos&#44; debemos añadir JavaScript externamente.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 Comunidad y recursos un poco más reducidos que los de frameworks grandes.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 El archivo CSS completo puede quedar <b className="notion-text-bold">pesado si no se modulariza o purga</b>
</li>
  </ul>
<p>
 Es un framework de<b className="notion-text-bold"> CSS </b>esta basado en <b className="notion-text-bold">Flexbox</b>
&#44; tiene modularidad y no requiere JavaScript. Su sintaxis semántica lo hace ideal para diseñar con claridad y eficiencia.
</p>
  
<h3 className="notion-heading_3 notion-color-default">
 Ventajas:
</h3>
  <ul className="notion-bulleted_list">
<li className="notion-bulleted_list_item notion-color-default">
 <b className="notion-text-bold">Tiene clases claras y legibles</b> como <code className="notion-text-code">.button</code>
&#44; <code className="notion-text-code">.title</code>
&#44; que facilitan la lectura del HTML.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 Basado en <b className="notion-text-bold">Flexbox</b>
&#44; lo que nos permite diseños responsivos fluidos y mobile‑first.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 Importamos únicamente los componentes que vamos a usar mediante Sass.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 No dependemos de JavaScript&#44; lo que lo hace muy ligero y compatible
</li>
  </ul><ul className="notion-bulleted_list">
<li className="notion-bulleted_list_item notion-color-default">
 No añade componentes interactivos: si necesitamos modales o dropdowns&#44; debemos añadir JS externo.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 El CSS completo puede ser pesado; conviene purgar lo innecesario.
</li>
  
<li className="notion-bulleted_list_item notion-color-default">
 No esta optimizado para navegadores antiguos &#40;ej. IE&#41;&#44; porque usa tecnologías modernas.
</li>
  </ul>

<h1 className="notion-heading_1 notion-color-default">
 Conclusiones
</h1>
  
<p>
 Los frameworks Utility‑First como Tailwind nos brindan control absoluto desde el HTML&#44; construyendo desde abajo hacia arriba con clases atómicas.
</p>
  
<p>
 En cambio&#44; los frameworks intermedios como Bulma nos dan componentes útiles listos para usar&#44; manteniendo flexibilidad sin imponernos un estilo rígido.
</p>
  
<p>
 Ambos enfoques tienen valor: uno se centra en el <b className="notion-text-bold">control total</b>
&#44; el otro en la <b className="notion-text-bold">productividad estructurada</b>.
</p>
  
<p>
 Por ello la clave es:
</p>
  <ul className="notion-bulleted_list">
<li className="notion-bulleted_list_item notion-color-default">
 Saber cuándo construir y cuándo usar lo que ya está construido.
</li>
  </ul>
